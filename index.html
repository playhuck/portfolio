Challenges

로그 파이프라인 시스템 재설계

주식회사 캐리버스

기존 단일 서버에서 로그 수집과 저장을 동시 처리하던 방식은 서버 병목과 장애 시 데이터 완전 유실 위험을 안고 있었습니다. 이를 해결하기 위해 Producer-Consumer 패턴으로 수집과 저장 프로세스를 독립적 서버로 분리하고, Redis Pub/Sub에서 Stream으로 전환하여 메시지 지속성과 순서를 보장하도록 개선했습니다. 결과적으로 유실되는 로그 없이 안정적인 로그 처리(유실률 0.5% 이하)를 달성했습니다.

기술 스택 : Redis Stream, AWS EKS, Docker, Spring Boot

실시간 대시보드 성능 최적화

주식회사 멘티스코

복잡한 JOIN 쿼리와 대용량 데이터로 인해 실시간 대시보드의 응답이 평균 10초 이상 지연되는 문제가 있었습니다. 일/주/월 단위 비정규화 테이블을 새롭게 설계하여 집계 쿼리 성능을 최적화하는 방식으로 접근했습니다. 이를 통해 대시보드 응답속도를 10초에서 1초 미만으로 대폭 개선할 수 있었습니다.

기술 스택 : Apache Superset, AWS Redshift, PostgreSQL

어드민 시스템 안정화

주식회사 캐리버스

런타임 타입 검증 부재로 5xx 에러가 빈발하고, 외부 서버별로 상이한 네이밍 컨벤션(BoxTape, boxTape, boxtape)으로 데이터 파싱이 실패하는 문제가 있었습니다. 제네릭 래퍼 클래스 기반 컴파일 타임 타입 검증 시스템을 구축하고, 외부 서버 응답을 camelCase로 통일하는 자동 변환 레이어를 구현했습니다. 이를 통해 런타임 에러 발생률을 대폭 감소시키고 외부 서버 연동 호환성 100%를 달성했습니다.

기술 스택 : Java 17, Spring Boot, Jackson, Generic Programming

환경변수 중앙화

주식회사 에딘트

개발과 운영 환경별로 분산된 환경변수로 인해 배포 오류와 협업 문제가 지속적으로 발생했습니다. AWS Secret Manager를 도입하여 환경변수를 중앙 집중화하고 버전 관리 체계를 구축했습니다. 이를 통해 환경변수 관련 배포 오류를 제로화하고 팀 협업 문제를 해결했습니다.
